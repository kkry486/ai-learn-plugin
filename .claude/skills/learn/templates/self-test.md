# 递进式自测模板

## 指令

基于以下概念的解释，生成三个递进层次的问题来检验用户的真实理解水平。

## 输入

【概念名】：{{concept}}
【费曼解释摘要】：{{feynman_summary}}
【用户的学习目标】：{{learning_goal}}

## 任务

生成三个层次的问题。每个问题的设计目标不同：

### 问题1 —— 理解层（验证是否听懂）

- 考察核心机制的理解
- 用户应能用 2-3 句话说清楚
- 设计思路：改变一个条件，问用户会发生什么变化
- 拒绝死记硬背型问题（如"XX有几个步骤？"）

### 问题2 —— 应用层（验证是否会做）

- 给出一个全新的具体场景，要求用户用该概念解决
- 场景必须有足够的上下文，让用户真正需要"操作"知识
- 不是问"怎么做"，而是给一个开放的小设计题

### 问题3 —— 边界层（验证是否知道局限）

- 考察对该概念局限性的理解
- 什么场景下这个概念不适用？
- 什么场景下这个概念会失效或产生反效果？
- 为什么这是根本性的限制（不是暂时的工程问题）？

## 输出格式

对每个问题，包含三个部分：

```
### 问题N：[层级名称]

**场景**：[问题背景和上下文]

**问题**：[具体的问题描述]

---

（用户回答后，进行以下点评）

**点评**：[对用户回答的具体评价，指出哪里好哪里不足]

**参考答案**：[一个高质量的参考回答]

**薄弱点标注**：[如果用户回答有误，标注薄弱点；如果正确，标注"已掌握"]
```
