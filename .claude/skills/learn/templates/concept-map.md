# 概念地图模板

## 指令

请完成以下分析任务。严格按照输出格式，不要添加额外说明文字。

## 输入

【用户想学的主题】：{{topic}}
【联网搜索结果】：{{search_results}}

## 任务

1. 将该主题拆解为 5-8 个核心子概念
2. 按学习依赖关系排序（必须先理解 A 才能理解 B、C）
3. 每个子概念用一句话说明它解决什么问题
4. 标注每个子概念的难度（入门 / 中等 / 进阶）

## 约束

- 子概念数量控制在 5-8 个，太少说明拆解不够，太多说明粒度太细
- 依赖关系必须正确——如果 B 依赖 A，A 必须在 B 之前
- "解决什么问题"必须从实际工程或认知需求出发，不能是"帮助理解X"这种废话
- 难度标注要真实——不要所有都标"中等"

## 输出格式（严格遵守）

```
## 概念地图

1. [概念名]（难度）→ 解决：[一句话说明它解决什么问题]
   ↓
2. [概念名]（难度）→ 解决：[一句话]
   ↓
3. [概念名]（难度）→ 解决：[一句话]
   ...
```
